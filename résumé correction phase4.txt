POINT D'ATTENTION : pg_hba.conf
--------------------------------------------------------------------------------
Constat : L'authentification PostgreSQL utilise "peer" par defaut pour les
          connexions locales. Cela signifie que seul l'utilisateur systeme
          "exemple" peut se connecter sans mot de passe.

          Pour PHP, la connexion fonctionne car pg_connect() utilise le socket
          local. Si on voulait se connecter a distance (depuis le client),
          il faudrait :
          1. Modifier pg_hba.conf : changer "peer" en "md5"
          2. Modifier postgresql.conf : listen_addresses = '*'
          3. Redemarrer PostgreSQL

          Pour notre projet, la connexion locale suffit (PHP tourne sur le
          meme serveur que PostgreSQL).


POINT D'ATTENTION : phpPgAdmin acces distant
--------------------------------------------------------------------------------
Constat : phpPgAdmin peut etre restreint a localhost ("Require local").
          Le script config_server.sh corrige automatiquement ce point en
          remplacant "Require local" par "Require all granted" pour
          permettre l'acces depuis le client.
